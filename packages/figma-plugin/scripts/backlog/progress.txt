
## 2026-03-27 - ImportPanel: $description and $scopes lost on import
- Bug: `mode.tokens.map()` in ImportPanel L263 only extracted `path`, `$type`, `$value`, silently dropping `$description` and `$scopes` that the controller reads from Figma variables (controller.ts L509-510).
- Fix: Include `$description` (when non-empty) and `$scopes` (when non-empty array) in the batch POST body. Also added `$scopes` to the batch endpoint's TypeScript type annotation.
- Files changed: `packages/figma-plugin/src/ui/components/ImportPanel.tsx` ~L263, `packages/server/src/routes/tokens.ts` ~L179
- **Learnings for future iterations:**
  - When mapping data for API calls, always check the source object for additional fields that should be forwarded — explicit destructuring/picking silently drops new fields added upstream.

## 2026-03-27 - ThemeManager `executeDeleteDimension` and `executeDeleteOption` don't check `res.ok`
- Added `res.ok` checks to `executeDeleteDimension`, `executeDeleteOption`, and `handleSetState`. On non-ok responses, error is shown via `setError()` and optimistic local state update is skipped (or rolled back for `handleSetState`).
- Files changed: `packages/figma-plugin/src/ui/components/ThemeManager.tsx` ~L230 (executeDeleteDimension), ~L279 (executeDeleteOption), ~L308 (handleSetState)
- **Learnings for future iterations:**
  - `fetch()` does NOT throw on HTTP error status codes — always check `response.ok`. This is the same recurring pattern across multiple backlog items.
  - Optimistic UI updates must be gated on `res.ok` or rolled back on failure; otherwise the UI temporarily lies to the user.

## 2026-03-27 - Command palette token results capped at 30 with no scroll-for-more
- Increased the cap from 30 to 100 results and added a "Showing 100 of N — refine your search to see more" footer when results are truncated.
- `filteredTokens` useMemo now returns `{ filteredTokens, totalTokenMatches }` so the UI can show total count without re-filtering.
- Files changed: `packages/figma-plugin/src/ui/components/CommandPalette.tsx` ~L96 (cap constant + useMemo), ~L280 (truncation footer)
- **Learnings for future iterations:**
  - When a list has a hard cap, always surface total match count alongside the shown count so users know results are truncated.
  - Destructuring from useMemo (`const { a, b } = useMemo(...)`) is a clean pattern for returning multiple related values without two separate memos.
