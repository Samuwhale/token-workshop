## 2026-03-27 - ThemeManager `executeDeleteDimension` and `executeDeleteOption` don't check `res.ok`
- Added `res.ok` checks to `executeDeleteDimension`, `executeDeleteOption`, and `handleSetState`. On non-ok responses, error is shown via `setError()` and optimistic local state update is skipped (or rolled back for `handleSetState`).
- Files changed: `packages/figma-plugin/src/ui/components/ThemeManager.tsx` ~L230 (executeDeleteDimension), ~L279 (executeDeleteOption), ~L308 (handleSetState)
- **Learnings for future iterations:**
  - `fetch()` does NOT throw on HTTP error status codes тАФ always check `response.ok`. This is the same recurring pattern across multiple backlog items.
  - Optimistic UI updates must be gated on `res.ok` or rolled back on failure; otherwise the UI temporarily lies to the user.

## 2026-03-27 - ImportPanel: `conflictPaths` state is not cleared when changing the target set тАФ old conflict list persists with the new set
- Added `setConflictPaths(null)` to the existing `useEffect` that fires on `targetSet` changes, making the clear authoritative regardless of how `targetSet` was changed
- Added `targetSetRef` (useRef) to track the current targetSet value; updated it in the same useEffect alongside the clear
- Fixed race condition in `handleImportStyles`: captured `checkingForSet = targetSet` at call time and discarded stale async results by checking `checkingForSet === targetSetRef.current` after the fetch resolves
- Files changed: `packages/figma-plugin/src/ui/components/ImportPanel.tsx` ~L79 (targetSetRef), ~L137 (useEffect update), ~L527 (race condition guard)
- **Learnings for future iterations:**
  - React state in async closures is always the value at call time тАФ use a `useRef` updated in a `useEffect` to always access the latest state value after an `await`
  - A `useEffect` that fires on state change is the most defensive place to clear derived state (like `conflictPaths`) тАФ covers all code paths including ones not yet written
