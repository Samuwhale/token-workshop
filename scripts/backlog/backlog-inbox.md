- [ ] Client URL encoding inconsistency — many handlers still use raw `${setName}/${tokenPath}` (no encodeURIComponent) for DELETE and other requests; if set names can contain `/`, these URLs would be misparsed. Audit and normalize.
- [ ] ImportPanel: 15-second timeout for reading Figma variables is too strict — for large files with many collections/modes, the read can legitimately exceed this window (e.g. 200+ variables on slower systems). Should extend to 30-45 seconds or make it configurable.
- [ ] ImportPanel: No visual distinction between disabled and enabled modes in variable import UI — checkboxes and text both fade to 40% opacity, making it hard to scan which modes are active at a glance; unchecked modes should have visible strikethrough or grayed label instead.
- [HIGH] ImportPanel: Handler for `styles-read` message does not clear `pendingSourceRef` before processing tokens — if the Figma plugin sends multiple `styles-read` responses (e.g. due to a race condition), the second response will be ignored because `pendingSourceRef` is still 'styles'. Should check and clear the ref at the top of the handler.
- [ ] ImportPanel: No validation that new set names don't conflict with existing sets — user can type a set name that already exists, then click "Create", and the input is simply treated as "select existing set" with no warning or feedback about the override behavior.
- [ ] ImportPanel: Styles read timeout does not have a `startReadTimeout()` call after `handleReadStyles` — variables read has a 15-second timeout but styles read does not, so if Figma hangs on reading styles, the UI never recovers (user is stuck in loading state indefinitely).
- [ ] ImportPanel: `conflictPaths` state is not cleared when changing the target set — if user fixes conflicts via "Revise selection", then switches to a different target set, `conflictPaths` should reset to null but does not (see line 775-781); old conflict list persists with the new set.
- [ ] ImportPanel: `readTimeoutRef` is not cleared in cleanup when component unmounts — the timer continues running in the background even after the panel is closed, and if it fires, it calls `setLoading(false)` on an unmounted component (memory leak / no-op but wasteful).
- [ ] ImportPanel: Figma Styles and JSON readers merge into single path normalization (replace `/` with `.`) but variables reader uses the same normalization — if a Figma variable is named `color/primary` and a style is named `color/primary`, they map to the same token path `color.primary`, creating silent conflicts that won't be detected until import.
- [HIGH] Build-time "Cannot access 'X' before initialization" TDZ errors — minifier (esbuild/Rollup) can hoist class or variable declarations into a temporal dead zone when circular imports or re-export patterns exist; any similar `ReferenceError: Cannot access 'Zo' before initialization` (or other mangled names) at runtime is caused by this. Audit circular dependencies with `madge --circular` and break cycles, or move shared types/values to a dedicated leaf module with no local imports.
