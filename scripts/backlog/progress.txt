## 2026-03-27 - Async side effects inside React setState updaters in useUndo
- Moved `slot.restore()` and `slot.redo!()` calls out of setState updater functions and into the callback body, reading `past`/`future` state directly. Added `past` and `future` as deps to `executeUndo`/`executeRedo` callbacks respectively. The `executingRef.current` guard prevents concurrent double-execution.
- Files changed: `packages/figma-plugin/src/ui/hooks/useUndo.ts` ~L27-54
- **Learnings for future iterations:**
  - setState updater functions must be pure â€” never call async side effects inside them. Read state outside the updater, then call setState with the derived value and run side effects after.
  - The `executingRef` guard is sufficient to prevent double-execution when `past`/`future` change rapidly; callbacks with those as deps are safe here.

## 2026-03-27 - `evalExpr` `**` operator precedence fix
- Tokenizer now emits `**` as a single token (2-char lookahead for `*`+`*`); `^` is normalized to `**`
- `parsePow` implemented with right-associative recursion so `**` binds tighter than `*`/`/`
- `**` handling removed from `parseMulDiv` â€” `parseMulDiv` now only handles `*` and `/`
- `2 * 3 ** 4` now correctly evaluates to 162 (was 1296); `^` also works as power alias
- Files changed: `packages/core/src/eval-expr.ts` ~L19-67
- **Learnings for future iterations:**
  - For multi-char operators (`**`), tokenize as a single token with lookahead â€” never handle them inside a lower-precedence parse function
  - Exponentiation is right-associative: `parsePow` recurses into itself for the RHS, not into `parseUnary`
