## 2026-03-27 - `updateAliasRefs` does not handle formula strings
- Replaced the exact-match/startsWith logic in both `updateAliasRefs` and `updateBulkAliasRefs` with regex-based global replacement. `updateAliasRefs` now uses a compiled `RegExp` with the escaped old path to match `{oldPath}` or `{oldPath.child}` anywhere in a string (including mid-formula like `"{spacing.base} * 2"`). `updateBulkAliasRefs` now uses a generic `/{([^}]+)}/g` regex and replaces each ref independently via the pathMap.
- Files changed: `packages/server/src/services/token-store.ts` ~L684, ~L728
- **Learnings for future iterations:**
  - The old guards (`s === exactMatch` / `s.startsWith(oldPrefix)`) only work for pure alias values. Formula strings embed refs mid-string, so regex with `g` flag + `String.replace` is required.
  - Escape special regex chars in path segments before inserting into a `RegExp` constructor — token paths can contain dots which are regex wildcards.
  - For `updateBulkAliasRefs` (rename single token), a simple `/{([^}]+)}/g` regex suffices because the pathMap has fully-qualified old→new mappings.
