## 2026-03-26 - Virtual scrolling for large token sets
- Added `flattenVisible()` helper that walks the token tree and returns `Array<{node, depth}>` respecting expand/collapse state
- Added `VIRTUAL_ITEM_HEIGHT = 28` and `VIRTUAL_OVERSCAN = 8` constants
- Added `virtualListRef` and `virtualScrollTop` state to `TokenList`
- Added `flatItems` useMemo (derives from `displayedTokens` + `expandedPaths`, disabled in table mode)
- Restructured render: toolbars moved into a `flex-shrink-0` wrapper; new `flex-1 overflow-y-auto` scroll container wraps only the token content area
- Tree render section replaced with virtual window: top/bottom padding spacers + slice of `flatItems`
- Added `skipChildren` prop to `TokenTreeNode`: when true, skips recursive children render (flat list owns child ordering)
- Added `onJumpToGroup` prop to `TokenTreeNode`: replaces DOM-based `scrollIntoView` with a callback that sets `virtualListRef.current.scrollTop`
- Added `useLayoutEffect` to scroll virtual list to highlighted token by index
- Removed `sticky` positioning from group headers (incompatible with flat virtual list)
- Files changed: `packages/figma-plugin/src/ui/components/TokenList.tsx` ~L50-75 (helpers), ~L172 (refs), ~L296 (effect), ~L741 (memo), ~L938-960 (handler + window vars), ~L964 (render restructure), ~L1282-1325 (virtual render), ~L1833 (TokenTreeNode props), ~L2005 (group header), ~L2159 (children guard)
- **Learnings for future iterations:**
  - Splitting toolbar (flex-shrink-0) from scrollable content (flex-1 overflow-y-auto) is the prerequisite for any virtual scroll in a panel that has both toolbars and a list
  - Virtual scroll with padding spacers (top/bottom `<div style={{height}}>`) is simpler than the translate3d approach and works well with React
  - `skipChildren` flag on the recursive tree component is the minimal change needed to convert recursive rendering to flat-list rendering — no separate component split required
  - Variable row heights (rows with descriptions) cause slight scroll position errors; overscan of 8 items compensates adequately for a plugin UI
  - `useLayoutEffect` (not `useEffect`) for scroll-to-highlighted prevents flash of wrong scroll position
  - Keyboard ↑/↓ navigation (`querySelectorAll('[data-token-path]')`) still works in virtual scroll since it only traverses visible DOM nodes — acceptable limitation for small navigation steps
