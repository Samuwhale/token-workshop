- On 409 conflict: instead of showing a small red error string, now sets `conflictExists` state and renders an inline "Token already exists. [Overwrite?]" message. The "Overwrite?" button calls `handleOverwriteToken` which PATCHes the existing token and shows a "Token overwritten" notification on success.
- Reset `conflictExists` whenever the name input changes so the conflict message clears as soon as the user edits the path.
- Imported `ApiError` from `apiFetch.ts` to reliably detect 409 vs other errors.
- Files changed: `packages/figma-plugin/src/ui/components/PropertyRow.tsx` ~L8, ~L83, ~L153, ~L513
- **Learnings for future iterations:**
  - `parent.postMessage({ pluginMessage: { type: 'notify', message } }, '*')` works from any UI component — no need to thread toast callbacks through props from App.tsx.
  - `ApiError` from `apiFetch.ts` has a `.status` field — use `err instanceof ApiError && err.status === 409` to branch on conflict vs other errors.
  - For overwrite flows, PATCH `/api/tokens/:set/*` accepts the same body fields as POST and updates in place.
