## 2026-03-28 - TokenEditor dependents section doesn't refresh after saving changes
- Patched `colorFlatMap` memo in TokenEditor to overlay the current editor's value for the token being edited, so dependent previews always reflect the latest saved/loaded value even when the parent's `allTokensFlat` is stale from an async refresh
- Root cause: after save, `handleEditorSave` calls `setEditingToken(null)` + `refreshAll()` (async). On reopen, `fetchToken()` gets fresh value but `allTokensFlat` prop may still be stale, causing `resolveRefValue` for dependents to use the old color
- Files changed: `packages/figma-plugin/src/ui/components/TokenEditor.tsx` ~L148-161
- **Learnings for future iterations:**
  - The `colorFlatMap` is derived from `allTokensFlat` (parent prop) but the editor fetches its own token value independently — these can be out of sync after save due to async parent refresh
  - Overlaying the editor's own value into derived maps is a general pattern for fixing stale-prop issues in editors that fetch their own data

## 2026-03-28 - `inferType` heuristic misclassifies composite tokens
- Tightened the `inferType` heuristic in `packages/core/src/resolver.ts` to require more distinctive key combinations instead of single-key matches:
  - **Shadow**: now requires `offsetX` or `offsetY` (previously any of `offsetX`/`offsetY`/`blur`/`spread` — so a composition with just `blur` was misclassified as shadow)
  - **Typography**: now requires `fontFamily` or `fontSize` (previously any of 5 typography keys)
  - **Border**: now requires `style` alongside `color` or `width` (previously `width`+`color` alone triggered border, which is ambiguous for compositions)
  - **Transition**: now requires `timingFunction` or at least 2 of 3 transition keys (previously any single key matched)
- Files changed: `packages/core/src/resolver.ts` ~L432-L448
- **Learnings for future iterations:**
  - When inferring composite token types from object keys, prefer keys that are *unique* to that type (e.g., `offsetX` for shadow, `fontFamily` for typography) over keys shared across types (e.g., `blur`, `color`, `width`).

  - `flattenWithNames` in `useTokens.ts` previously dropped `$extensions` entirely — only `$value`, `$type`, and `$name` were preserved in `TokenMapEntry`
  - Tokens without scopes pass all filters (opt-in restriction model), maintaining backward compatibility
  - The `SCOPE_TO_PROPERTIES` map bridges Figma's variable scope vocabulary to the plugin's `BindableProperty` type
