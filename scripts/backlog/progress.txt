  6. **Template literal cases** in `MetadataEditor.tsx`, `TokenEditor.tsx`, `TokenList.tsx`, `BatchEditor.tsx`: manually updated conditional class strings
  7. **`ThemeManager.tsx`**: Fixed `focus:ring-1` → `focus-visible:ring-1` on div role=list elements
  8. **`PropertyPicker.tsx`**: Fixed non-standard `focus:border-[var(--color-figma-border-selected)]` → `focus-visible:border-[var(--color-figma-accent)]`
  9. **Transparent search inputs** (`HistoryPanel.tsx`, `history/GitCommitsSource.tsx`): Removed `focus:outline-none` — these inputs rely on the global CSS rule

- **Learnings for future iterations:**
  - In Tailwind 4 (`@import "tailwindcss"`), utilities are in `@layer utilities`. Unlayered CSS in `styles.css` has higher cascade priority and will override `outline-none` on matching selectors — this makes global focus-visible CSS rules very reliable even when Tailwind classes try to suppress them
  - The most efficient pattern for a sweeping focus-style fix: (1) add global CSS rule as the foundation, (2) use sed batch replacement for the common `outline-none focus:border-` pattern, (3) manually fix template-literal conditional class strings, (4) fix div-based interactive widgets (sliders, listboxes) that aren't covered by element-type selectors
  - Transparent inputs embedded in styled containers (search boxes) should NOT have `focus:outline-none` unless the container provides a `focus-within:` indicator — remove the suppression and let the global rule provide the ring
