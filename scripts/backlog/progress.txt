## 2026-04-03 - Multiple UI hooks clear state optimistically before async API calls complete
- Fixed three hooks that cleared state before async operations completed, causing inconsistent UI on network errors.
- `useDragDrop.ts`: Moved `setDragSource(null)` from before the API call to after it (both on success and error paths) in `handleDropOnGroup` and `handleDropReorder`. Previously tokens "vanished" during the async gap since the drag state was cleared optimistically.
- `useGroupOperations.ts`: Added `groupOpInProgress` ref to prevent concurrent move/copy operations from interleaving. Both `handleConfirmMoveGroup` and `handleConfirmCopyGroup` now guard against re-entrancy — a second confirmation click while the first API call is in-flight is silently ignored.
- `useTokenSyncBase.ts`: Changed `applyDiff` to only clear rows/dirs for successfully applied tokens. On partial failure, failed rows remain visible in the sync table so the user can see exactly which tokens failed and retry. Previously `setRows([])` and `setDirs({})` were called unconditionally before computing whether there were failures.
- Files changed: `packages/figma-plugin/src/ui/hooks/useDragDrop.ts` ~L62, ~L89, ~L97, ~L147; `packages/figma-plugin/src/ui/hooks/useGroupOperations.ts` ~L44, ~L126, ~L149; `packages/figma-plugin/src/ui/hooks/useTokenSyncBase.ts` ~L237
- **Learnings for future iterations:**
  - For drag operations, `setDragSource(null)` controls the drag-active visual state — clearing it before the API call makes tokens appear to have moved successfully before the server confirms, causing a confusing snap-back on failure. Always clear it in both success and error paths after the async call.

## 2026-04-04 - Git sync conflict resolution route has no rollback, no input validation, and no partial-failure handling
- Added `resolveAllConflicts(resolutions)` method to `GitSync` service that:
  1. Validates each resolution entry has a non-empty `file` string and a `choices` object.
  2. Validates all choice values are `'ours'` or `'theirs'`.
  3. Path-validates all files (directory traversal check).
  4. Cross-validates each file against the live git conflict state — returns 400 if a file is not currently conflicted.
  5. Validates all region indices are within range for each file — returns 400 for out-of-range indices.
  6. Resolves all file contents in memory before touching the filesystem.
  7. Writes and stages each resolved file; on any failure, rolls back already-staged files to their conflicted state via `git checkout -m -- <file>`.

- **Learnings for future iterations:**
  - `git checkout -m -- <file>` re-checks out a file with its original merge conflict markers, making it the correct rollback primitive for a partially-resolved merge state.
  - Resolving all file contents in memory before any `fs.rename`/`git add` side effects eliminates the main partial-failure window — failures during in-memory computation never touch git state.

- Added `OPEN_TOKEN_SEARCH: '⌘⇧F'` to `SHORTCUT_KEYS` and a corresponding registry entry in the Global group of `SHORTCUT_REGISTRY`.
- Added `initialQuery?: string` prop to `CommandPalette` — used as the initial `useState` value so the input is pre-populated on mount.
- Added `commandPaletteInitialQuery` state in `App.tsx` (`useState('')`); set to `'>'` when Cmd+Shift+F fires, reset to `''` on Cmd+K.
- The shortcut toggles the palette (consistent with other global shortcuts); closing still works via Escape.
- Files changed: `packages/figma-plugin/src/ui/shared/shortcutRegistry.ts` ~L51, ~L93; `packages/figma-plugin/src/ui/components/CommandPalette.tsx` ~L88, ~L149; `packages/figma-plugin/src/ui/App.tsx` ~L336, ~L730, ~L3170
- **Learnings for future iterations:**
  - The `showCommandPalette && <CommandPalette />` mounting pattern means `useState(initialQuery)` naturally resets on every open — no effect or key trick needed.
  - The stable callback ref pattern for keyboard shortcuts (`keyboardShortcutRef.current = fn`) reads current state on every render, so new state variables (`commandPaletteInitialQuery`) are safely accessible inside the handler without extra deps.
  - Resetting `commandPaletteInitialQuery` on Cmd+K is necessary; otherwise a prior Cmd+Shift+F open would leave `>` as the initial query for all future Cmd+K opens.
