## 2026-04-04 - Git sync conflict resolution route has no rollback, no input validation, and no partial-failure handling
- Added `resolveAllConflicts(resolutions)` method to `GitSync` service that:
  1. Validates each resolution entry has a non-empty `file` string and a `choices` object.
  2. Validates all choice values are `'ours'` or `'theirs'`.

- **Learnings for future iterations:**
  - `git checkout -m -- <file>` re-checks out a file with its original merge conflict markers, making it the correct rollback primitive for a partially-resolved merge state.
