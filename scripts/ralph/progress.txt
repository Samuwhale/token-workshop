## Codebase Patterns
- Plugin UI is all in `packages/figma-plugin/src/ui/` ‚Äî App.tsx, hooks/, components/
- `controller.ts` has pre-existing Figma environment type errors (figma global not available in tsc) ‚Äî ignore these, they are expected
- Build command: `cd packages/figma-plugin && npm run build` (uses vite via build.mjs)
- Typecheck: `npx tsc --noEmit -p packages/figma-plugin/tsconfig.json` ‚Äî ignore controller.ts errors
- Tailwind CSS variable pattern: `var(--color-figma-*)` for all colors (accent, text, bg, border, etc.)
- State management: local React state (no Redux/Zustand), hooks in `ui/hooks/`
- Server API: REST at `serverUrl` (default `http://localhost:9400`), endpoints: `/api/sets`, `/api/tokens/:set`, `/api/sync/*`, `/api/health`
- Token tree structure: leaf nodes have `$value`, group nodes are plain objects; keys starting with `$` are metadata
- `DELETE /api/tokens/:set/:path` deletes the key at that path ‚Äî works for both leaf tokens AND group paths (deletes entire subtree)
- Alias format is `{path.to.token}` (curly braces, dot-separated path), detected via `isAlias()` in resolveAlias.ts
- `ConfirmModal` component at `components/ConfirmModal.tsx` ‚Äî reusable for all destructive confirmations (danger prop, Escape/outside-click cancel)
- `useUndo` hook at `hooks/useUndo.ts` ‚Äî single undo slot with 8s toast; use `pushUndo(UndoSlot)` after any destructive op; exports `UndoSlot` type
