- [ ] usePinnedTokens hook exists but pinning UI is not wired in TokenList — `usePinnedTokens.ts` already has the infrastructure (pin/unpin/isPinned) but `TokenList.tsx` never calls it; add a pin button to token rows (visible on hover) and a "Pinned" section at the top of the list so power users can surface their most-used tokens
- [ ] Token group expand/collapse state is lost on page reload — expand state is saved to `sessionStorage` in `TokenList.tsx` which is cleared when the panel is closed or Figma restarts; switch to `localStorage` so users don't have to re-expand the same groups every session
- [ ] No batch delete for selected tokens — select mode in `TokenList.tsx` allows multi-select but provides no batch delete action; users must delete tokens one at a time even when many are selected; add a "Delete selected (N)" button in the selection toolbar with a single confirmation dialog showing the count and any alias dependents
- [ ] Lint violations not shown inline on token list rows — `useLint.ts` hook data is available in `TokenList.tsx` but violations are only surfaced in the AnalyticsPanel tab; add a small warning indicator on each token row that has an active lint violation so users can see issues without leaving the token list
- [ ] No suppress/ignore action on individual lint violations in AnalyticsPanel — `suppressedKeys` state already exists in `AnalyticsPanel.tsx` but there is no button to suppress a specific violation; add a per-row dismiss/suppress button so users can acknowledge known issues and reduce noise without disabling the whole rule
- [ ] ImportPanel does no upfront schema validation before importing — user can upload invalid JSON or wrong-format files and the error only surfaces mid-import with a generic message; validate the file structure (check for `$type`/`$value` fields or known nesting format) immediately after file selection and show a clear error before the user clicks Import
- [ ] No bulk-fill action for tokens missing from multiple theme sets — when a token exists in a source set but is missing from several enabled sets, the user must open each set individually to add the value; add a "Fill missing in all sets" shortcut in the ThemeManager coverage view that copies the source value into every set that lacks the token
- [ ] Theme dimension rename does not validate against existing names — renaming a dimension to a name already in use is only rejected by the server after the request is sent; check for name collisions client-side in `ThemeManager.tsx` and show an inline error before the rename is submitted
- [ ] Variable/style sync shows no per-item progress during large syncs — `useVariableSync` and `useStyleSync` only toggle a loading spinner; for syncs involving hundreds of tokens the panel sits blank for many seconds; emit incremental progress counts from the plugin sandbox message handler and display "Syncing N / M variables…" in the PublishPanel
- [ ] Success and error toasts disappear with no way to review them — `useToastStack` auto-dismisses after ~3 s and there is no persistent notification log; users who missed a message have no way to see it again; add a small notification history accessible from a bell/clock icon in the panel header, showing the last ~10 messages with timestamps
- [ ] WCAG contrast ratio not shown inline in color picker — `ContrastChecker.tsx` exists as a separate panel but there is no live contrast feedback while the user is picking a color; add a small contrast badge (AA/AAA/Fail) inside the color picker that updates in real-time against a configurable background swatch, making it actionable where the decision is being made
- [ ] Theme coverage gaps not prominently surfaced — coverage data is fetched in `ThemeManager.tsx` but missing tokens are buried inside collapsible sections; add a summary badge on each theme option (e.g. "12 missing") visible at a glance, and a "Show missing only" filter so users can quickly identify and fill gaps without scrolling through all tokens
